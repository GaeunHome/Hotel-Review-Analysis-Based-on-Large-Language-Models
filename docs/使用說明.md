# 使用說明

## 執行流程

### IPA 分析（主要流程）

```bash
# 1. 排序原始資料
python -m hotel_ipa.preparation.sequence

# 2. GPT-4o-mini 逐條分析（提取關鍵詞 + 情感 + 評分 + 重要度）
python -m hotel_ipa.analysis.gpt_4o_mini_ipa

# 3. 兩階段 AI 分類（關鍵詞 → 12 標準屬性）
python -m hotel_ipa.classification.classify

# 4. 生成 IPA 統一儀表板（含 Post-hoc 重要度 + 動態趨勢）
python -m hotel_ipa.visualization.ipa_dashboard

# 5. 統計檢定（可選）
python -m hotel_ipa.stats.ipa_priority_tests
```

### SWOT 分析

```bash
# 在 IPA 流程完成後執行
python -m hotel_ipa.swot.swot_engine
```

### 穩定性驗證 / Post-hoc 重要度

```bash
python -m hotel_ipa.validation.stability
python -m hotel_ipa.importance.post_hoc
```

## IPA 分析概念

### 改進優先度公式

```
改進優先度 = 重要度（全局） × 績效缺口
績效缺口 = 5 - 平均績效
```

- 重要度為全局值（所有酒店整體平均），反映顧客普遍重視程度
- 績效為各酒店獨立計算

### IPA 四象限

| 象限 | 條件 | 策略 |
|------|------|------|
| I 繼續保持 | 高重要度 + 高績效 | 維持競爭優勢 |
| II 集中改善 | 高重要度 + 低績效 | 優先投入資源 |
| III 過度投入 | 低重要度 + 高績效 | 可調配資源 |
| IV 低優先 | 低重要度 + 低績效 | 有餘力再改善 |

### 12 個標準屬性

地理位置、性價比、服務、房間、餐廳、停車、清潔度、公共設施、周邊環境、衛浴、交通、睡眠品質

## SWOT 判定規則 (R1-R8)

基於 Wu et al. (2024)，比較焦點酒店與競爭酒店的內部優劣勢與績效差異：

| 規則 | 焦點 | 競爭 | 績效比較 | SWOT |
|------|------|------|----------|------|
| R1 | 優勢 | 優勢 | 焦點 >= 競爭 | S |
| R2 | 優勢 | 優勢 | 焦點 < 競爭 | T |
| R3 | 優勢 | 劣勢 | 焦點 >= 競爭 | O |
| R4 | 優勢 | 劣勢 | 焦點 < 競爭 | T |
| R5 | 劣勢 | 劣勢 | 焦點 < 競爭 | W |
| R6 | 劣勢 | 劣勢 | 焦點 >= 競爭 | O |
| R7 | 劣勢 | 優勢 | 焦點 < 競爭 | T |
| R8 | 劣勢 | 優勢 | 焦點 >= 競爭 | O |

## 統計檢定

### ipa_priority_tests.py

對 IPA 彙總統計資料進行檢定：
- 改進優先度分布分析
- 重要度與績效差距檢定（配對 t 檢定 / Wilcoxon）
- 相關性分析（Pearson / Spearman）
- 常態性檢定（Shapiro-Wilk）

### statistical_tests.py

對原始評論資料進行完整統計檢定：
- 描述性統計、常態性檢定
- 配對樣本 t 檢定、ANOVA
- 非參數檢定（Wilcoxon、Kruskal-Wallis）
- 差距分析

### 判讀要點

- **p <= 0.05**: 差異顯著
- **Cohen's d**: |d| < 0.2 小效果，0.5 中效果，>= 0.8 大效果
- 統計顯著不等於實務重要，需同時考慮效果量
